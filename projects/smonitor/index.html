Small

The session monitor consists of two small applications. The agent is ~80KB big and the size of the administrator is ~2MB. Both of them are standard JEE web applications. On the server you want to track you only need to install the agent application.

Simple

The concept of the application is simple. It can be used to analyse the application session and track user activity in your application. You need to install the administrator on the server, which can be connected to each agent application.

Fast

The agent and administrator are designed for small memory usage and fast response time. You can get quick overview of the parameters and size of the HTTP-Sessions, which is stored in the web-container. All configuration files are created runtime

Download

Download the administrator and agent application. Current version: 1.0.0-Alpha1

  • JBoss EAP 6.1.0 - Session monitor admin
  • JBoss EAP 6.1.0 - Session monitor agent

Deploy

Now deploy the agent application to all the servers you want to track. Then deploy the administator application on the server from which you are going to track all the applications and servers.

Both applications can run on the same server. For the localhost server open the link: http://localhost:8080/smonitor.

Registration of agent

First you need to register each agent. Go to the Agents menu and add an agent.

Click on the add agent button. Add the agent parameters like the name and address. For the localhost agent the address would be http://localhost:8080. Only the active and correctly running agents are used for tracking the sessions.

Search for session

Go to the Sessions menu. Select search criteria and click on the search button.

Select session and click on the session menu to refresh, delete or get more information of the HTTP-session.
