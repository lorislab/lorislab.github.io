Easy to integrate

This application provides the maven plugin for easy integration. For non maven project you need to check the ARM properties file to build your project correctly.

Dashboard

The dashboard of the application give you an overview about the systems and the deployed version on it. For each component you can get more information like source code changes.

Notification

The monitor can automatically check the version on your systems and send a notification to users about the new deployed version. For non reachable system, the developer can register a new version for the systems by using the maven plugin.

Maven integration

Add the org.lorislab.armonitor:armonitor-maven-plugin:{{version}} plugin with the goal build to your MAVEN project. 

<plugin>
	<groupId>org.lorislab.armonitor</groupId>
	<artifactId>armonitor-maven-plugin</artifactId>
	<version>{{version}}</version>
	<executions>
		<execution>
			<phase>prepare-package</phase>
			<goals>
				<goal>build</goal>
			</goals>
			<configuration>
				<releaseBuild>RC1</releaseBuild>
				<releaseScm>rev123</releaseScm>
				<custom>
					<key1>value1</key1>
					<key2>value2</key2>
				</custom>
			</configuration>
		</execution>
	</executions>
</plugin>

The plugin has few attributes to manage the build release information. The maven parameters like groupId, artifactId, version and build time are store automatically.

The plugin parameters
Name Default value Description
releaseBuild The release build number.
releaseScm The revision number.
custom The map of custom parameters.

JBoss7 service

For JBoss7 we provided the service that can get the versions and ARM properties from your application deployed on the server. You can download the service manually from our release repository armonitor-agent-web.war or get it through maven. 

<dependency>
	<groupId>org.lorislab.armonitor</groupId>
	<artifactId>armonitor-agent-web</artifactId>
	<packaging>war</packaging>
	<version>{{version}}</version>
</dependency>

The URL to register the service in the monitor is http://localhost:8080/armonitor

EAR or WAR services

For non JBoss7 application server or if you cannot deploy the service as a WAR application, you can add the ARM service directly to your application by putting these maven dependencies to WAR project. armonitor-agent-rs.jar armonitor-agent-impl.jar 

<dependency>
	<groupId>org.lorislab.armonitor</groupId>
	<artifactId>armonitor-agent-rs</artifactId>
	<version>{{version}}</version>
</dependency>
<dependency>
	<groupId>org.lorislab.armonitor</groupId>
	<artifactId>armonitor-agent-impl</artifactId>
	<version>{{version}}</version>
</dependency>

The URL to register the service in the monitor is http://localhost:8080/my-app-context/my-rs-servlet
